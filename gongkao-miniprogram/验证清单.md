# 微信小程序 - 验证清单

## 构建 npm 后检查

### 1. 文件结构检查
在项目根目录应该能看到：
```
gongkao-miniprogram/
├── node_modules/          ✅ 已创建
│   └── @vant/weapp/      ✅ 已安装
└── miniprogram_npm/       ⏳ 构建npm后生成
    └── @vant/
        └── weapp/
```

### 2. 页面加载检查

打开小程序模拟器，检查首页是否正常显示：
- [ ] 用户信息卡片显示
- [ ] 打卡按钮显示
- [ ] 今日课程卡片显示
- [ ] Vant 组件（按钮、图标、标签）正常显示

### 3. Console 检查

打开调试器 Console，不应该出现以下错误：
- ❌ `Component is not found in path`
- ❌ `module "@vant/weapp/button/index" is not defined`

### 4. 如果页面空白

可能原因和解决方案：

| 问题 | 解决方案 |
|-----|---------|
| miniprogram_npm 未生成 | 重新执行「构建 npm」 |
| 组件路径错误 | 检查 app.json 中的 usingComponents |
| 缓存问题 | 清除编译缓存 → 重新编译 |

## 快速测试

在 Console 中输入：
```javascript
// 测试1: 检查 app 实例
getApp()

// 测试2: 检查当前页面
getCurrentPages()[0]

// 测试3: 检查全局数据
getApp().globalData.baseUrl
// 应该输出: "https://shxtj.chaim.top/api/v1"
```

## 如果一切正常

应该看到：
- ✅ 登录页面显示完整（LOGO、功能介绍、登录按钮）
- ✅ Vant 按钮样式正确
- ✅ Console 无错误

可以开始正常开发了！
