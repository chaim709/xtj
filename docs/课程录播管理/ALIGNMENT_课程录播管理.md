# 对齐文档 - 课程录播管理功能

## 文档信息
- **创建日期**：2026-01-28
- **文档版本**：v1.0
- **任务阶段**：Align（对齐阶段）

---

## 1. 原始需求

用户需求：在学员管理系统中添加课程录播网址和记录功能，每次上课完成后记录腾讯会议的录播链接。

## 2. 需求理解

### 2.1 核心功能
- 记录每天课程的录播链接（腾讯会议录播）
- 关联到课表（Schedule），支持区分上午/下午/晚间时段
- 完整记录信息：链接、标题、日期、科目、老师、时长、备注

### 2.2 展示场景
1. **班次详情页** - 新增"录播"标签页，展示该班次所有录播
2. **学员详情页** - 展示学员所在班次的录播列表
3. **录播管理菜单** - 课程管理下新增独立入口

### 2.3 项目上下文
- 现有 `Schedule` 模型已包含日期、科目、上午/下午/晚间老师信息
- 录播应与 `Schedule` 关联，一个课表日期可有多个录播（上午、下午、晚间）
- 复用现有的科目、老师数据

## 3. 边界确认

### 3.1 范围内
- 录播记录的增删改查
- 三处展示位置的UI实现
- 与现有课表、班次、学员的关联

### 3.2 范围外
- 视频上传/存储功能（只存链接）
- 视频播放器集成
- 录播权限控制（本期不做）

## 4. 技术约束

- 技术栈：Flask + SQLAlchemy + Jinja2 + Bootstrap 5
- 数据库：SQLite
- 复用现有架构模式和代码风格

## 5. 验收标准

1. 可以在班次下添加/编辑/删除录播记录
2. 录播记录包含：链接、标题、日期、科目、老师、时长、备注、时段
3. 班次详情页新增"录播"标签页
4. 学员详情页可查看其所在班次的录播
5. 课程管理下有独立的"录播管理"入口
